The `roop.core` module provides the main entry point and orchestration logic for the face swapping application.

## Main Functions

### run()

Main entry point that orchestrates the entire face swapping process.

```python
def run() -> None
```

**Description**

Executes the complete workflow:
1. Parses command-line arguments
2. Performs pre-flight checks
3. Validates frame processors
4. Limits system resources
5. Starts processing (headless or UI mode)

**Example**

```python
import roop.core

# Run the application
roop.core.run()
```

### start()

Starts the face swapping processing workflow.

```python
def start() -> None
```

**Description**

Handles the actual processing logic:
- For images: Processes image-to-image face swapping
- For videos: Extracts frames, processes each frame, and creates output video
- Manages FPS preservation and audio restoration based on global settings

**Processing Flow**

1. **Image-to-Image**: Copies target to output and applies frame processors
2. **Image-to-Video**: 
   - Creates temporary directory
   - Extracts frames from target video
   - Processes each frame with face swapping
   - Creates output video with optional FPS/audio preservation

**Example**

```python
import roop.globals
import roop.core

# Configure paths
roop.globals.source_path = 'source.jpg'
roop.globals.target_path = 'target.mp4'
roop.globals.output_path = 'output.mp4'

# Start processing
roop.core.start()
```

**Command-Line Arguments**

<ParamField path="-s, --source" type="string">
  Path to source image containing the face to swap
</ParamField>

<ParamField path="-t, --target" type="string">
  Path to target image or video where face will be swapped
</ParamField>

<ParamField path="-o, --output" type="string">
  Path to output file or directory
</ParamField>

<ParamField path="--frame-processor" type="list[string]" default="['face_swapper']">
  Frame processors to use (e.g., face_swapper, face_enhancer)
</ParamField>

<ParamField path="--keep-fps" type="boolean" default="false">
  Preserve original video FPS
</ParamField>

<ParamField path="--keep-audio" type="boolean" default="true">
  Preserve original audio from target video
</ParamField>

<ParamField path="--keep-frames" type="boolean" default="false">
  Keep temporary frames after processing
</ParamField>

<ParamField path="--many-faces" type="boolean" default="false">
  Process every face in the frame instead of just one
</ParamField>

<ParamField path="--video-encoder" type="string" default="libx264">
  Video encoder to use. Choices: `libx264`, `libx265`, `libvpx-vp9`
</ParamField>

<ParamField path="--video-quality" type="int" default="18">
  Output video quality (0-51, lower is better)
</ParamField>

<ParamField path="--max-memory" type="int" default="4 (macOS) or 16 (other)">
  Maximum RAM usage in GB
</ParamField>

<ParamField path="--execution-provider" type="list[string]" default="['cpu']">
  Execution providers for inference (e.g., cpu, cuda, rocm)
</ParamField>

<ParamField path="--execution-threads" type="int" default="1 or 8">
  Number of execution threads
</ParamField>

**Example**

```bash
python -m roop.core \
  --source face.jpg \
  --target video.mp4 \
  --output result.mp4 \
  --keep-fps \
  --execution-provider cuda
```

## Helper Functions

### limit_resources()

Limits system resource usage to prevent memory issues.

```python
def limit_resources() -> None
```

**Description**

Configures resource limits:
- Sets TensorFlow GPU memory limit to 1024 MB to prevent memory leaks
- Applies OS-specific memory limits based on `roop.globals.max_memory`
- Handles platform-specific implementations (macOS, Windows, Linux)

**Platform Behavior**

- **macOS**: Uses `max_memory * 1024^6` bytes
- **Windows**: Uses `SetProcessWorkingSetSize` via ctypes
- **Linux**: Uses `resource.setrlimit` with `RLIMIT_DATA`

**Example**

```python
import roop.globals
import roop.core

roop.globals.max_memory = 8  # 8 GB
roop.core.limit_resources()
```
